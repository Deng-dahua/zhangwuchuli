# 涉税风险规则质量审计

## 概述

本技能提供涉税风险规则库（`static/tax_risk_rules_default.json`）的完整质量检查流程。
每次新增、修改、或合并规则后，都应运行本检查确保规则库健康。

## 触发条件

- 用户说"检查规则"、"审计规则"、"规则去重"、"规则质量"、"归类整理"
- 新增规则（不论来源：手动输入、PDF提取、AI生成）
- 合并两个来源的规则后
- 修改分类归属后

## 工作流

### 第1步：运行审计脚本

```bash
python .workbuddy/skills/tax-risk-rules-audit/scripts/audit_rules.py static/tax_risk_rules_default.json
```

该脚本自动执行以下8层检查：

| 层 | 检查项 | 阈值 | 说明 |
|----|--------|------|------|
| 1 | ID/名称精确去重 | - | 完全相等的 ID 或 item 名 |
| 2 | 名称相似度 | ≥85% | 可能同一规则不同表述 |
| 3 | detail相似度 | ≥80% | 同分类+跨分类 |
| 4 | 语义同类扫描 | - | 10组关键词跨分类扫描，报告同一主题跨分类散布情况 |
| 5 | 碎片分类检测 | <2条 | 单条规则独自分类的建议合并 |
| 6 | 归类不当检测 | - | 税种关键词出现在异常分类中 |
| 7 | level字段一致性 | - | 高→高风险、中→中风险、低→低风险 |
| 8 | 评分跨度检查 | ≥5分 | 同分类内评分差异过大 |

### 第2步：分析输出

脚本会先在每层输出检查结果，最后输出汇总。解读规则：

- **第1层**：任何 ❌ 都必须修复（精确重复是bug）
- **第2-3层**：相似度高的需人工判断是否应合并
- **第4层**：语义同类跨分类散布是信息性的，不一定是问题（如"进项转出"的6条规则分别覆盖不同场景：不及时、免税、走逃企业、转入转出等）
- **第5层**：碎片分类建议合并，但像"印花税"这种独立税种可保留
- **第6层**：多数是误报（城建税必然含增值税、个税规则必然含个税关键词），需逐条人工判断
- **第7层**：直接修复
- **第8层**：含"良好"评分（0分）的分类自然跨度大，正常

### 第3步：修复问题

根据检查结果，使用以下模式修复：

**合并重复规则**：
```python
# 保留较完善的规则，删除重复的，将差异内容合并到保留规则中
data = [r for r in data if str(r['id']) != '要删除的ID']
for r in data:
    if str(r['id']) == '保留的ID':
        r['detail'] = '合并后的完整detail'
        r['suggestion'] = '合并后的完整suggestion'
```

**合并碎片分类**：
```python
moves = {
    "规则名称": "目标分类",
}
for r in data:
    if r["item"] in moves:
        r["category"] = moves[r["item"]]
```

**规范 level 字段**：
```python
level_map = {"高": "高风险", "中": "中风险", "低": "低风险"}
for r in data:
    if r["level"] in level_map:
        r["level"] = level_map[r["level"]]
```

### 第4步：再次验证

修复后重新运行审计脚本，确认所有修复生效、无新问题引入。

### 第5步：提交

更新 `static/index.html` 中的版本号，git commit + push。

## 关键规则

- **语义同类≠重复**：第4层找到的跨分类散布是正常的，不同角度检测同一风险主题（如"进项转出"可以从"不及时""免税收入""走逃企业"等不同维度切入）
- **碎片分类可保留**：像"印花税""房产税""城建税"等独立税种，即使只有1-2条规则也可保留为独立分类，方便未来扩展
- **归类不当误报**：第6层基于关键词匹配，必然产生大量误报。需人工逐条判断，不要盲目移动

## 脚本参数

```bash
python scripts/audit_rules.py [json_path]
```

- 默认路径：`static/tax_risk_rules_default.json`
- 可指定其他 JSON 文件路径
