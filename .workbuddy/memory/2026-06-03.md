# 2026-06-03 工作日志

## 修复：页面切换 DOM 污染（全力整改）

### 问题
序时账、总账、明细账、利润表、资产负债表、科目余额表全部出现页面互相污染——点序时账显示科目余额表内容，点总账显示其他模块内容。

### 根因
`navigateTo(page)` 创建了独立的 `#page-xxx` 容器并传给 `renderXxx(container)`，但所有 `renderXxx()` 函数定义没有接收 `container` 参数，而是直接 `document.getElementById('content-area')` 操作全局容器，导致各页面 DOM 互相覆盖。

### 修复过程
1. 从干净 commit `3e7cad2` 恢复 `static/index.html`（之前 Python 脚本修复引入了 `\'` 非法 JS 语法）
2. 使用 **Node.js 脚本**（而非 Python）执行替换，避免 Python 字符串转义问题
3. 修复内容：
   - 22 个 `renderXxx()` 函数签名全部加上 `container` 参数
   - 18 处 `const el = document.getElementById('content-area')` 改为 `container || document.getElementById('content-area')`
   - 7 处残留的 `document.getElementById('content-area').innerHTML` 改为 `el.innerHTML`
   - `showCompanyManager` 同步修复签名、el 定义、内部调用传参
   - 总账表删除「期初余额」「期初方向」两列（表头 + 数据行）
4. JS 语法验证通过（`new Function(js)` 无异常）

### Commit
- `d9d57f7` fix: 彻底修复页面切换DOM污染问题

### 教训
修改 JS 文件时绝不能用 Python 字符串替换——Python 的 `\'` 转义会被写入文件成为非法 JS 语法。应使用 Node.js 脚本或 `Edit` 工具。
